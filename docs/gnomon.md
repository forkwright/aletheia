# Gnomon

*γνώμων - that which reveals by indicating*

A sundial's gnomon doesn't contain time. It casts a shadow that makes time legible. The best names work the same way - they don't describe what something is. They reveal what was already there.

This document is the naming system behind Aletheia and everything adjacent to it. It is itself a gnomon: it indicates the system without containing it.

---

## I. The Medium

### Why Greek

Ancient Greek was engineered across centuries for a single purpose - distinguishing between modes of knowing. Where English has "knowledge," Greek has ἐπιστήμη (episteme - systematic knowledge), γνῶσις (gnosis - direct acquaintance), τέχνη (techne - craft knowledge), φρόνησις (phronesis - practical wisdom), and νοῦς (nous - direct apprehension). These aren't synonyms. They're fundamentally different stances toward the world.

This precision matters. When you name a project *Dianoia*, you're not saying "thinking" - you're saying *discursive reasoning that works through problems step by step*, as distinct from noesis (immediate insight) or episteme (systematic knowledge). The Greek carries the distinction natively. English requires a paragraph.

Greek also offers:

- **Productive compounding.** Roots, prefixes, and suffixes combine systematically. μετα- (beyond) + ξύν (together) + νόησις (understanding) = μεταξυνόησις - understanding-beyond-togetherness. The grammar builds meaning.
- **Defamiliarization.** An unfamiliar word resists assumption. You have to ask what it means, which means you have to think about what the thing *does*. This is a feature.
- **Density.** Meaning compressed into sound. Ἀλήθεια is four syllables carrying an entire ontology - truth as *unconcealment*, the negation of forgetting.

The goal is not erudition. The goal is precision tools for a system that distinguishes between many closely related things.

### Pronunciation

Classical reconstructed conventions. Enough to speak clearly, not enough to satisfy a philologist.

| Symbol | Sound | Example |
|--------|-------|---------|
| η (eta) | "ay" as in *say* | Ἀλήθεια → ah-LAY-thay-ah |
| ει | "ay" | Ποίησις → poy-AY-sis |
| αι | "eye" | - |
| οι | "oy" | Ποιητική → poy-ay-ti-KAY |
| ου | "oo" | Μουσεῖον → moo-SAY-on |
| θ (theta) | "th" as in *think* | Θέμις → THEH-mis |
| χ (chi) | "k" or guttural "ch" | - |
| φ (phi) | "f" | Φήμη → FAY-may |

Stress typically falls on the penultimate syllable for words ending in -ια/-εια.

## II. The Method

### Building Names

Greek names are constructed, not found. Three components combine:

**Roots** carry the semantic core - the fundamental action or concept:

| Root | From | Meaning |
|------|------|---------|
| γνω- (gno-) | γιγνώσκω | knowing, recognizing |
| νο- (no-) | νοῦς | perceiving, apprehending |
| σκοπ- (skop-) | σκοπέω | watching, examining |
| θεωρ- (theor-) | θεωρέω | contemplating, beholding |
| ποι- (poi-) | ποιέω | making, creating |
| ἐργ- (erg-) | ἔργον | working, functioning |
| λεγ-/λογ- (leg-/log-) | λέγω | gathering, reasoning |
| ἴχν- (ichn-) | ἴχνος | tracing, tracking |
| φαν- (phan-) | φαίνω | showing, bringing to light |

**Prefixes** modify relationship - how the root acts:

| Prefix | Meaning | Effect |
|--------|---------|--------|
| συν- (syn-) | together, with | Collective action |
| ἀνα- (ana-) | up, again, throughout | Recurrence, ascent |
| ἐπι- (epi-) | upon, toward | Direction, application |
| δια- (dia-) | through, across | Penetration, process |
| μετα- (meta-) | beyond, after, change | Transcendence, shift |
| προσ- (pros-) | toward, attending to | Orientation, care |
| αὐτο- (auto-) | self | Reflexivity |

**Suffixes** determine what kind of thing the name denotes:

| Suffix | Denotes | Example |
|--------|---------|---------|
| -ια/-εια | Abstract state or quality | ἀλήθεια (truth), σοφία (wisdom) |
| -σις | Process, act of doing | ποίησις (making), νόησις (understanding) |
| -τική | Art, skill, capacity | ῥητορική (rhetoric), ποιητική (poetics) |
| -μα | Result of an action | σχῆμα (form), γράμμα (letter) |
| -ων/-μων | Agent, one who does | γνώμων (one who indicates) |

A name like **προσοχή** (prosoche) is: πρός (toward) + ἔχω (to hold) → *holding-toward* → sustained attention. The construction tells you what it is.

### The Layer Test

Every name must work at multiple levels of abstraction simultaneously. This is the primary quality test.

| Layer | What it answers | Failure mode |
|-------|----------------|--------------|
| **L1 - Practical** | What does this do? What is it for? | Too abstract to use |
| **L2 - Structural** | How does this relate to other things? What role does it play? | Isolated, doesn't compose |
| **L3 - Philosophical** | What stance toward knowing does this embody? | Decorative, not meaningful |
| **L4 - Reflexive** | Does the name exhibit what it describes? | Inert, doesn't reward reflection |

A name that works at L1 but not L3 is a label. A name that works at L3 but not L1 is pretension. A name that works at all relevant layers is *dimensionally resonant* - equally true at every level of abstraction, opening rather than closing as you look closer.

**Example - Aletheia (ἀλήθεια):**

| Layer | Reading |
|-------|---------|
| L1 | A software system - agent ecosystem, memory, orchestration |
| L2 | The substrate that makes the other systems possible |
| L3 | Truth as unconcealment - ἀ-λήθεια, the negation of λήθη (forgetting, concealment). Not truth as correspondence but truth as *revealing what was hidden* |
| L4 | The system itself practices unconcealment - surfacing latent knowledge, making hidden patterns legible, refusing to let things stay forgotten |

**Example - Dianoia (διάνοια):**

| Layer | Reading |
|-------|---------|
| L1 | The planning system - project decomposition, phased execution |
| L2 | The structured thinking that connects vision to implementation |
| L3 | Discursive reasoning - thinking *through* (δια-) problems step by step, as distinct from noesis (direct apprehension). Plato's divided line: dianoia is the mode between opinion and pure understanding |
| L4 | A planning system named for step-by-step reasoning that itself works by stepping through phases - the name describes its own operation |

When a name fails the layer test, you've usually misidentified what you're naming. The name isn't wrong - your understanding of the thing is incomplete. Go back.

## III. The Theory

### Modes of Attention

You are not naming things. You are not even naming essences.

You are naming **modes of attention** - ways of seeing that make certain kinds of knowing possible.

This distinction matters because it changes what a name does. A label points at an object. A mode-of-attention name points at a *stance* - the particular orientation of mind that a domain requires. The domain doesn't exist independently of how you attend to it. A library attended to as inventory is different from a library attended to as curation. The mode of attention constitutes the domain.

This is why the names feel right when they work. **Prosoche** (sustained attention) isn't a label on a daemon process. It names the fundamental act the daemon performs - and the fundamental act required of anyone building systems that *pay attention rather than perform it*. The name doesn't describe the software. It describes the stance the software embodies.

This is why the names resist simplification. **Mouseion** isn't "media server." It's the mode of attention in which you relate to collected knowledge as a living resource rather than archived data - the original μουσεῖον was a place of study, not storage. Calling it a media server kills what it actually is.

And this is why getting the name wrong feels wrong in a way that goes beyond preference. A misnamed thing has been *misattended to*. You've identified the wrong stance, which means you'll build it wrong, use it wrong, relate to it wrong. The name is upstream of everything.

### Dimensional Resonance

A name exhibits dimensional resonance when it is equally true at every level of abstraction without straining at any of them.

This is not cleverness. It's a diagnostic. If a name requires effort to read at L3, the philosophical layer is forced - you picked a Greek word that sounds right rather than one that *means* right. If L1 doesn't land immediately, you've prioritized depth over function. Genuine resonance is effortless because the name has correctly identified a mode of attention that *actually operates at all those layers simultaneously*.

Resonance also means the layers reinforce rather than contradict. Aletheia's L1 (a software system) doesn't fight its L3 (truth as unconcealment) - the software literally unconceals. This coherence across layers is what makes a name feel inevitable rather than assigned.

The test is simple: explain the name at L1 to someone who doesn't care about Greek philosophy. Then explain it at L3 to someone who does. If both explanations feel honest - neither dumbed down nor dressed up - the name resonates.

### Topology

Names in a system are not independent. They compose.

The relationships between names carry meaning. **Noesis** (direct apprehension) and **Dianoia** (discursive reasoning) aren't just two agent-adjacent concepts - they're the two sides of Plato's divided line, the upper segment of knowing. One sees the whole immediately; the other works through it step by step. Placing them in the same system asserts that both modes are necessary and that they complement rather than compete.

**Prosoche** (sustained attention) is the *practice* that makes **Aletheia** (unconcealment) possible. You can't reveal what's hidden without attending carefully. The names encode this dependency - prosoche is infrastructure for aletheia, and naming them this way makes that relationship legible.

Topological coherence means:
- **Containment is meaningful.** Aletheia contains Dianoia, Prosoche, the agents. This isn't organizational convenience - it's the claim that truth-as-unconcealment requires both structured reasoning and sustained attention.
- **Adjacency is meaningful.** Names at the same level should share a mode-of-attention register. Agent names (Syn, Demiurge, Syl, Akron) all name *what the agent does as a stance*, not what domain it covers.
- **Absence is meaningful.** What you don't name stays unconstituted. Choosing not to name something is choosing not to attend to it as a distinct mode.

When you add a new name to the system, it must compose with what exists. A name that fits locally but creates dissonance in the topology has identified the right mode of attention in isolation but the wrong one in context.

## IV. The Philosophy

### Naming as Epistemology

The naming process is not downstream of understanding. It is constitutive of it.

You cannot name something well until you understand what it is. But you do not fully understand what it is until the name arrives. This is not a contradiction - it's a hermeneutic circle. The search for the name *is* the process of clarification. Each candidate you consider and reject teaches you something about what the thing isn't, which sharpens what it is, which narrows the name, which deepens the understanding.

This is why naming too early produces brittle names - you haven't yet understood the mode of attention. And why forcing a name feels dishonest - you're asserting an understanding you don't have. The name should arrive with a feeling of recognition, not invention. *Of course* the planning system is Dianoia. *Of course* the truck is Akron. The name was always there; you just had to understand enough to see it.

This epistemological function is why the system uses Greek rather than invented words. An invented word carries only what you put into it. A Greek word carries what centuries of philosophical use have deposited. When you name something *prosoche*, you inherit the Stoic tradition of attention as moral practice - and that inheritance *teaches you something about your own system that you didn't put there*. The name is smarter than the namer. The tradition fills in layers you hadn't consciously intended.

### The Gnomon Principle

A gnomon (γνώμων) is, at its simplest, the vertical element of a sundial. It doesn't contain time. It doesn't even measure time. It casts a shadow, and the shadow makes time legible.

The best names are gnomons. They indicate without containing. They reveal what was latent in the thing by casting it into relief - making the implicit explicit, the hidden visible. The name Aletheia doesn't *make* the system true. It reveals that the system was already an act of unconcealment. The name Dianoia doesn't *make* the planner reason discursively. It reveals that phased planning always was discursive reasoning.

This is the deepest function of the naming system: not labeling, not even identifying modes of attention, but *unconcealing what things already are*. The gnomon adds nothing to the sundial. It merely makes legible what the sun was always doing.

A name succeeds as a gnomon when someone encounters it, learns what it means, and feels that the thing has become more itself - not differently described but more clearly *seen*.

A name fails as a gnomon when the name draws attention to itself rather than to what it names. If you're explaining the name, it's decoration. If the name is explaining the thing, it's a gnomon.

## V. The Practice

### When Naming

1. **Wait.** Let the thing reveal its nature before fixing the name. Build first, name when the mode of attention becomes clear. Premature naming produces labels.

2. **Identify the mode of attention.** Not what the thing *is* but how you *attend* to it. What stance does engaging with it require? What way of seeing does it enable?

3. **Find the core action.** Greek roots are verbs and processes, not static nouns. What does this thing *do*? What does it *reveal*? The root should name an action.

4. **Construct the name.** Use the prefix-root-suffix system. Let the grammar do work - the construction should be meaningful, not just euphonic.

5. **Run the layer test.** L1 through L4. If any layer strains, the name is wrong - go back to step 2, because you've probably misidentified the mode of attention.

6. **Check the topology.** Does this name compose with what exists? Does it create coherence or dissonance? Does adjacency and containment remain meaningful?

7. **Listen for recognition.** The right name feels discovered, not invented. If you're arguing for why it fits, it doesn't.

### Anti-Patterns

**Naming the implementation.** You named what it does technically, not how you attend to it. "Tracker" is an implementation. Prosoche is a mode of attention that tracking serves.

**Naming too precisely.** A name that works at only one layer is brittle. If the implementation changes, the name breaks. Mode-of-attention names survive refactors because the stance persists even when the mechanism changes.

**Forcing the Greek.** If you have to work through three dictionaries to justify the connection, the name is wrong. The right Greek word should feel like it was waiting. If no Greek word fits, the mode of attention isn't clear yet.

**Naming for audience.** If the name exists to impress rather than to indicate, it's pretension. The test: would you use this name alone, in your own notes, with no one watching? If not, it's performance.

**Ignoring topology.** A name that sounds right in isolation but conflicts with adjacent names disrupts the system. The system is more important than any individual name.

## Appendix: Sources

The intellectual roots of this system, in case the names send you looking:

- **Plato** - *Meno* (anamnesis), *Republic* (the divided line: dianoia and noesis), *Theaetetus* (knowledge and perception)
- **Aristotle** - *Nicomachean Ethics* (ergon, phronesis, techne), *Politics* (chrematistike), *Metaphysics* (energeia)
- **The Stoics** - Epictetus and Marcus Aurelius on prosoche (attention as practice), autarkeia (self-sufficiency)
- **Heidegger** - *Being and Time* on aletheia as unconcealment (though the personal use of the word predates reading him)
- **Jacob Klein** - *Greek Mathematical Thought and the Origin of Algebra* on the relationship between naming and knowing
- **Liddell-Scott-Jones** - *Greek-English Lexicon*, the definitive reference for classical Greek
