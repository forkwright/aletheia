# Gnomon

*γνώμων - that which reveals by indicating*

A sundial's gnomon doesn't contain time. It casts a shadow that makes time legible. The best names work the same way - they don't describe what something is. They reveal what was already there.

This document is the naming system behind Aletheia and everything adjacent to it. It is itself a gnomon: it indicates the system without containing it.

---

## I. The Medium

### Why Greek

Ancient Greek was engineered across centuries for a single purpose - distinguishing between closely related things that matter. Where English has "knowledge," Greek has ἐπιστήμη (episteme - systematic knowledge), γνῶσις (gnosis - direct acquaintance), τέχνη (techne - craft knowledge), φρόνησις (phronesis - practical wisdom), and νοῦς (nous - direct apprehension). These aren't synonyms. They're fundamentally different stances toward the world.

But this precision isn't limited to epistemics. Where English has "form," Greek has μορφή (morphe - the form that makes a thing *this kind* of thing), σχῆμα (schema - the shape or figure of a thing), εἶδος (eidos - the visible form, the look of a thing), and τύπος (typos - the impression left by a blow). Where English has "order," Greek has τάξις (taxis - arrangement), κόσμος (kosmos - the beautiful order of a whole), and λόγος (logos - the rational principle that structures). The language is equally precise about *what things are* as it is about *how you know them*.

This precision matters. When you name a project *Dianoia*, you're not saying "thinking" - you're saying *discursive reasoning that works through problems step by step*, as distinct from noesis (immediate insight) or episteme (systematic knowledge). When you name a module *Taxis*, you're not saying "organization" - you're saying *the specific arrangement that makes a collection coherent rather than merely grouped*. The Greek carries the distinction natively. English requires a paragraph.

Greek also offers:

- **Productive compounding.** Roots, prefixes, and suffixes combine systematically. μετα- (beyond) + ξύν (together) + νόησις (understanding) = μεταξυνόησις - understanding-beyond-togetherness. The grammar builds meaning.
- **Defamiliarization.** An unfamiliar word resists assumption. You have to ask what it means, which means you have to think about what the thing *does*. This is a feature.
- **Density.** Meaning compressed into sound. Ἀλήθεια is four syllables carrying an entire ontology - truth as *unconcealment*, the negation of forgetting.

The goal is not erudition. The goal is precision tools for a system that distinguishes between many closely related things.

### Pronunciation

Classical reconstructed conventions. Enough to speak clearly, not enough to satisfy a philologist.

| Symbol | Sound | Example |
|--------|-------|---------|
| η (eta) | "ay" as in *say* | Ἀλήθεια → ah-LAY-thay-ah |
| ει | "ay" | Ποίησις → poy-AY-sis |
| αι | "eye" | - |
| οι | "oy" | Ποιητική → poy-ay-ti-KAY |
| ου | "oo" | Μουσεῖον → moo-SAY-on |
| θ (theta) | "th" as in *think* | Θέμις → THEH-mis |
| χ (chi) | "k" or guttural "ch" | - |
| φ (phi) | "f" | Φήμη → FAY-may |

Stress typically falls on the penultimate syllable for words ending in -ια/-εια.

## II. The Method

### Building Names

Greek names are constructed, not found. Three components combine:

**Roots** carry the semantic core - the fundamental action or concept:

| Root | From | Meaning | Domain |
|------|------|---------|--------|
| γνω- (gno-) | γιγνώσκω | knowing, recognizing | epistemic |
| νο- (no-) | νοῦς | perceiving, apprehending | epistemic |
| σκοπ- (skop-) | σκοπέω | watching, examining | epistemic |
| θεωρ- (theor-) | θεωρέω | contemplating, beholding | epistemic |
| μνη- (mne-) | μνήμη | remembering, recalling | epistemic |
| ποι- (poi-) | ποιέω | making, creating | active |
| ἐργ- (erg-) | ἔργον | working, functioning | active |
| τεκ- (tek-) | τέκτων | building, crafting | active |
| λεγ-/λογ- (leg-/log-) | λέγω | gathering, reasoning | active |
| ἴχν- (ichn-) | ἴχνος | tracing, tracking | active |
| φαν- (phan-) | φαίνω | showing, bringing to light | active |
| κρι- (kri-) | κρίνω | separating, judging | active |
| ζητ- (zet-) | ζητέω | seeking, inquiring | active |
| μορφ- (morph-) | μορφή | forming, shaping | structural |
| σχη- (sche-) | σχῆμα | figuring, patterning | structural |
| ταξ- (tax-) | τάξις | arranging, ordering | structural |
| ἁρμ- (harm-) | ἁρμονία | fitting together, joining | structural |
| ὁρ- (hor-) | ὅρος | bounding, delimiting | structural |
| ὑλ- (hyl-) | ὕλη | material, substrate | structural |

The domain labels (epistemic, active, structural) are guides, not fences. Many names cross domains - μελέτη (melete) is an epistemic practice enacted through active repetition. The root table is vocabulary, not taxonomy.

**Prefixes** modify relationship - how the root acts:

| Prefix | Meaning | Effect |
|--------|---------|--------|
| συν- (syn-) | together, with | Collective action |
| ἀνα- (ana-) | up, again, throughout | Recurrence, ascent |
| ἐπι- (epi-) | upon, toward | Direction, application |
| δια- (dia-) | through, across | Penetration, process |
| μετα- (meta-) | beyond, after, change | Transcendence, shift |
| προσ- (pros-) | toward, attending to | Orientation, care |
| αὐτο- (auto-) | self | Reflexivity |

**Suffixes** determine what kind of thing the name denotes:

| Suffix | Denotes | Example |
|--------|---------|---------|
| -ια/-εια | Abstract state or quality | ἀλήθεια (truth), αὐτάρκεια (self-sufficiency) |
| -σις | Process, act of doing | ποίησις (making), νόησις (understanding) |
| -τική | Art, skill, capacity | ῥητορική (rhetoric), ποιητική (poetics) |
| -μα | Result of an action | σχῆμα (form), γράμμα (letter) |
| -ων/-μων | Agent, one who does | γνώμων (one who indicates), τέκτων (craftsman) |
| -ος/-ον | Thing, instrument, place | ὄργανον (tool), σημεῖον (sign) |
| -ή | Abstract quality or practice | μελέτη (practice), προσοχή (attention) |

A name like **προσοχή** (prosoche) is: πρός (toward) + ἔχω (to hold) → *holding-toward* → sustained attention. The construction tells you what it is.

### The Layer Test

Every name must work at multiple levels of abstraction simultaneously. This is the primary quality test.

| Layer | What it answers | Failure mode |
|-------|----------------|--------------|
| **L1 - Practical** | What does this do? What is it for? | Too abstract to use |
| **L2 - Structural** | How does this relate to other things? What role does it play? | Isolated, doesn't compose |
| **L3 - Philosophical** | What essential nature does this name unconceal? | Decorative, not meaningful |
| **L4 - Reflexive** | Does the name exhibit what it describes? | Inert, doesn't reward reflection |

A name that works at L1 but not L3 is a label. A name that works at L3 but not L1 is pretension. A name that works at all relevant layers is *dimensionally resonant* - equally true at every level of abstraction, opening rather than closing as you look closer.

**Example - Aletheia (ἀλήθεια):**

| Layer | Reading |
|-------|---------|
| L1 | A software system - agent ecosystem, memory, orchestration |
| L2 | The substrate that makes the other systems possible |
| L3 | Truth as unconcealment - ἀ-λήθεια, the negation of λήθη (forgetting, concealment). Not truth as correspondence but truth as *revealing what was hidden* |
| L4 | The system itself practices unconcealment - surfacing latent knowledge, making hidden patterns legible, refusing to let things stay forgotten |

**Example - Dianoia (διάνοια):**

| Layer | Reading |
|-------|---------|
| L1 | The planning system - project decomposition, phased execution |
| L2 | The structured thinking that connects vision to implementation |
| L3 | Discursive reasoning - thinking *through* (δια-) problems step by step, as distinct from noesis (direct apprehension). Plato's divided line: dianoia is the mode between opinion and pure understanding |
| L4 | A planning system named for step-by-step reasoning that itself works by stepping through phases - the name describes its own operation |

**Example - Taxis (τάξις):**

| Layer | Reading |
|-------|---------|
| L1 | Agent scaffolding and orchestration - creates workspaces, arranges configuration |
| L2 | The structural layer that gives each agent its place within the system |
| L3 | The Greek concept of proper arrangement - not mere ordering but the specific placement that makes a whole coherent. A phalanx has taxis; a crowd does not |
| L4 | The module arranges agents into formation. It IS the act of τάξις - each workspace it creates is a position assigned within the larger order |

**Example - Symbolon (σύμβολον):**

| Layer | Reading |
|-------|---------|
| L1 | Authentication - JWTs, sessions, password hashes |
| L2 | The gate-check that pylon consults before admitting requests |
| L3 | In antiquity, a symbolon was a clay tablet split between parties - proof of identity by showing the matching half. The module enacts this ancient practice with modern cryptography |
| L4 | Every JWT the module issues IS a symbolon. The split-token practice is not a metaphor for what the module does - it is literally what the module does |

When a name fails the layer test, you've usually misidentified what you're naming. The name isn't wrong - your understanding of the thing is incomplete. Go back.

## III. The Theory

### What Gnomon Names

A gnomon unconceals. What it unconceals is whatever the shadow falls on.

The first things we named were **modes of attention** - ways of seeing that make certain kinds of knowing possible. Prosoche (sustained attention), melete (disciplined practice), noesis (direct apprehension), dianoia (discursive reasoning). This was natural because the system is cognitive. Its primary concern is how agents attend to the world.

But the shadow falls further than that. As we named more of the system, we found things that aren't stances at all - they're **structures** (taxis: arrangement, koina: shared commons), **artifacts** (symbolon: the split-token itself, semeion: the sign), **practices** (melete bridges both - it names the discipline and the disciplined act), **instruments** (organon: the tool set). These passed the layer test with the same ease. They exhibited dimensional resonance. They composed topologically. They were gnomon names - they just weren't modes of attention.

The recognition: **modes of attention** were the first domain gnomon named, not the boundary of what gnomon can name. The method is more general than its first application. What unifies gnomon names isn't that they all describe stances. What unifies them is that they all **unconceal essential natures** - the fundamental character of a thing that persists across implementation changes, that operates at every layer simultaneously, that was always there waiting to be seen.

A mode of attention is one kind of essential nature. A structural form is another. An artifact's identity is another. The layer test doesn't ask "what stance does this embody?" as a restriction - it asks "what does this name unconceal?" The answer might be an epistemic stance (prosoche), a compositional form (taxis), a relational practice (symbolon), or a purposive orientation (telos). The test is the same. The depth required is the same. The dimensional resonance is the same.

This is not an expansion. It's a recognition. The naming system was always doing this. We just described it too narrowly at first because we started with attention. Now the description catches up with the practice.

### Domains of Naming

Names in the system tend to unconceal natures from different domains. These are not categories to assign - they're lenses for understanding what a name does. Many names operate across domains simultaneously.

**Epistemic** — How you attend. The stance that makes a kind of knowing possible.

Prosoche (sustained attention), melete (disciplined practice), theoros (observation without intervention), noesis (direct apprehension), dianoia (discursive reasoning), zetesis (inquiry, seeking).

These name the *relationship between knower and known*. They're the most natural domain for a cognitive system because the system's primary act is knowing.

**Structural** — What something is formally. The compositional form that makes a thing coherent.

Taxis (arrangement into formation), koina (shared commons), harmonia (fitting together of parts), schema (the figure or pattern).

These name *how parts compose into wholes*. They're essential for infrastructure, data architecture, and any module whose nature is organizational rather than epistemic.

**Artifactual** — What gets produced. The identity of things the system creates or maintains.

Symbolon (the split-token), semeion (the sign or signal), organon (the tool), mneme (the memory store).

These name *what a thing IS as an object* - not how you use it or attend to it, but its essential character as an artifact. A symbolon is a symbolon whether you're issuing it or verifying it; the name describes the artifact, not the act.

**Teleological** — What something aims toward. The end that gives purpose to process.

Telos (the final cause, the aim), autarkeia (self-sufficiency as a property), aletheia (unconcealment as ongoing practice).

These name *what orients action*. They appear where a thing's essential nature is its directedness - where what it IS is inseparable from what it's FOR.

The boundaries are soft by design. Melete is epistemic (a way of knowing through practice) AND active (the practice itself). Symbolon is artifactual (the token as object) AND relational (the split creates a bond between parties). Taxis is structural (arrangement) AND teleological (arrangement *toward* coherence). Names that span domains aren't failing the taxonomy - they're demonstrating the dimensional resonance that makes them work.

### Dimensional Resonance

A name exhibits dimensional resonance when it is equally true at every level of abstraction without straining at any of them.

This is not cleverness. It's a diagnostic. If a name requires effort to read at L3, the philosophical layer is forced - you picked a Greek word that sounds right rather than one that *means* right. If L1 doesn't land immediately, you've prioritized depth over function. Genuine resonance is effortless because the name has correctly identified an essential nature that *actually operates at all those layers simultaneously*.

Resonance also means the layers reinforce rather than contradict. Aletheia's L1 (a software system) doesn't fight its L3 (truth as unconcealment) - the software literally unconceals. This coherence across layers is what makes a name feel inevitable rather than assigned.

The test is simple: explain the name at L1 to someone who doesn't care about Greek philosophy. Then explain it at L3 to someone who does. If both explanations feel honest - neither dumbed down nor dressed up - the name resonates.

### Topology

Names in a system are not independent. They compose.

The relationships between names carry meaning. **Noesis** (direct apprehension) and **Dianoia** (discursive reasoning) aren't just two agent-adjacent concepts - they're the two sides of Plato's divided line, the upper segment of knowing. One sees the whole immediately; the other works through it step by step. Placing them in the same system asserts that both modes are necessary and that they complement rather than compete.

**Prosoche** (sustained attention) is the *practice* that makes **Aletheia** (unconcealment) possible. You can't reveal what's hidden without attending carefully. The names encode this dependency - prosoche is infrastructure for aletheia, and naming them this way makes that relationship legible.

**Symbolon** (split-token authentication) and **Autarkeia** (self-sufficiency/portability) are adjacent acts of identity - one proves who you are in the moment, the other preserves who you are across time. The names make this structural pairing visible in a way that "auth" and "portability" never could.

**Mneme** (memory store) and **Melete** (disciplined practice) compose as process: mneme holds what was experienced, melete refines it into wisdom. The names reveal that storage and distillation are not independent functions but two phases of the same act - remembering well requires both keeping and examining.

Topological coherence means:
- **Containment is meaningful.** Aletheia contains Dianoia, Prosoche, the agents. This isn't organizational convenience - it's the claim that truth-as-unconcealment requires both structured reasoning and sustained attention.
- **Adjacency is meaningful.** Names at the same level should share a register of essential nature. Module names (symbolon, melete, taxis, organon) all unconceal what the module *essentially is*, not what domain it covers.
- **Absence is meaningful.** What you don't name stays unconstituted. Choosing not to name something is choosing not to attend to it as having a distinct essential nature.

When you add a new name to the system, it must compose with what exists. A name that fits locally but creates dissonance in the topology has identified the right essential nature in isolation but the wrong one in context.

## IV. The Philosophy

### Naming as Epistemology

The naming process is not downstream of understanding. It is constitutive of it.

You cannot name something well until you understand what it is. But you do not fully understand what it is until the name arrives. This is not a contradiction - it's a hermeneutic circle. The search for the name *is* the process of clarification. Each candidate you consider and reject teaches you something about what the thing isn't, which sharpens what it is, which narrows the name, which deepens the understanding.

This is why naming too early produces brittle names - you haven't yet understood the essential nature. And why forcing a name feels dishonest - you're asserting an understanding you don't have. The name should arrive with a feeling of recognition, not invention. *Of course* the planning system is Dianoia. *Of course* the truck is Akron. The name was always there; you just had to understand enough to see it.

This epistemological function is why the system uses Greek rather than invented words. An invented word carries only what you put into it. A Greek word carries what centuries of philosophical use have deposited. When you name something *prosoche*, you inherit the Stoic tradition of attention as moral practice - and that inheritance *teaches you something about your own system that you didn't put there*. The name is smarter than the namer. The tradition fills in layers you hadn't consciously intended.

### The Gnomon Principle

A gnomon (γνώμων) is, at its simplest, the vertical element of a sundial. It doesn't contain time. It doesn't even measure time. It casts a shadow, and the shadow makes time legible.

The best names are gnomons. They indicate without containing. They reveal what was latent in the thing by casting it into relief - making the implicit explicit, the hidden visible. The name Aletheia doesn't *make* the system true. It reveals that the system was already an act of unconcealment. The name Dianoia doesn't *make* the planner reason discursively. It reveals that phased planning always was discursive reasoning. The name Taxis doesn't *make* the scaffolder arrange things. It reveals that workspace creation was always an act of placing-into-formation.

This is the deepest function of the naming system: **unconcealing what things already are**. Not labeling, not categorizing, not even identifying modes of attention as a privileged domain. The gnomon adds nothing to the sundial. It merely makes legible what the sun was always doing.

What the shadow falls on varies. Sometimes it falls on how you attend (prosoche). Sometimes on what something is structurally (taxis). Sometimes on the artifact a process produces (symbolon). Sometimes on the aim that gives a thing its purpose (telos). The gnomon doesn't choose what to illuminate. You point it, and the shadow reveals whatever was there.

A name succeeds as a gnomon when someone encounters it, learns what it means, and feels that the thing has become more itself - not differently described but more clearly *seen*.

A name fails as a gnomon when the name draws attention to itself rather than to what it names. If you're explaining the name, it's decoration. If the name is explaining the thing, it's a gnomon.

## V. The Practice

### When Naming

1. **Wait.** Let the thing reveal its nature before fixing the name. Build first, name when the essential nature becomes clear. Premature naming produces labels.

2. **Identify what needs unconcealing.** Not what the thing *does technically* but what it *essentially is*. This might be a stance (how you attend), a form (how it composes), an artifact (what it produces), or a purpose (what it aims toward). The domain doesn't matter. The depth does.

3. **Find the core action.** Greek roots are verbs and processes, not static nouns. What does this thing *do*? What does it *reveal*? The root should name an action or fundamental quality.

4. **Construct the name.** Use the prefix-root-suffix system. Let the grammar do work - the construction should be meaningful, not just euphonic.

5. **Run the layer test.** L1 through L4. If any layer strains, the name is wrong - go back to step 2, because you've probably misidentified the essential nature.

6. **Check the topology.** Does this name compose with what exists? Does it create coherence or dissonance? Does adjacency and containment remain meaningful?

7. **Listen for recognition.** The right name feels discovered, not invented. If you're arguing for why it fits, it doesn't.

### Anti-Patterns

**Naming the implementation.** You named what it does technically, not what it essentially is. "Tracker" is an implementation. Prosoche is the essential nature that tracking serves. "Portability" is an outcome. Autarkeia is the essential property that makes portability possible.

**Naming too precisely.** A name that works at only one layer is brittle. If the implementation changes, the name breaks. Essential-nature names survive refactors because the nature persists even when the mechanism changes.

**Forcing the Greek.** If you have to work through three dictionaries to justify the connection, the name is wrong. The right Greek word should feel like it was waiting. If no Greek word fits, the essential nature isn't clear yet - which means you don't understand the thing well enough to name it.

**Naming for audience.** If the name exists to impress rather than to indicate, it's pretension. The test: would you use this name alone, in your own notes, with no one watching? If not, it's performance.

**Ignoring topology.** A name that sounds right in isolation but conflicts with adjacent names disrupts the system. The system is more important than any individual name.

**Restricting the shadow.** If you find yourself saying "this thing can't have a gnomon name because it's not a mode of attention," you've confused the first application with the method itself. The question isn't "is this a stance?" It's "does this thing have an essential nature that Greek can unconceal?" If yes, name it. If the layer test passes, it belongs.

## Appendix: Sources

The intellectual roots of this system, in case the names send you looking:

- **Plato** - *Meno* (anamnesis), *Republic* (the divided line: dianoia and noesis), *Theaetetus* (knowledge and perception), *Phaedrus* (form and essence)
- **Aristotle** - *Categories* (the fundamental ways a thing can be), *Nicomachean Ethics* (ergon, phronesis, techne), *Politics* (chrematistike), *Metaphysics* (energeia, morphe/hyle, telos)
- **The Stoics** - Epictetus and Marcus Aurelius on prosoche (attention as practice), autarkeia (self-sufficiency), melete (care as disciplined practice)
- **Heidegger** - *Being and Time* on aletheia as unconcealment (though the personal use of the word predates reading him)
- **Jacob Klein** - *Greek Mathematical Thought and the Origin of Algebra* on the relationship between naming and knowing
- **Liddell-Scott-Jones** - *Greek-English Lexicon*, the definitive reference for classical Greek
