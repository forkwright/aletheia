# Phase 2.1 Plan 01: Fix addMemories Session Wiring Summary

Thread `session_id` through `MemoryFlushTarget.addMemories` so distillation-extracted facts reach Qdrant instead of silently failing with HTTP 400.

## What Was Built

Phase 2-02 (INTG-06) enforced `session_id` on `/add_batch`. The `addMemories()` implementation in `aletheia.ts` was never updated to include it, meaning every distillation memory flush returned 400 and all extracted facts were silently dropped. This plan closes that gap with four targeted edits across the call chain, plus test updates.

| 2 | Update test assertions for new sessionId parameter | df11700 |

## Key Changes

### hooks.ts
- `MemoryFlushTarget.addMemories`: added required `sessionId: string` third parameter
- `flushToMemory`: added required `sessionId: string` parameter after `optsOrRetries` (no default — empty string is falsy in Python and would trigger a 400)
- Internal `target.addMemories` call: now forwards `sessionId`

### aletheia.ts
- `addMemories` implementation: accepts `sessionId: string` third parameter
- POST body to `/add_batch`: includes `session_id: sessionId`

### pipeline.ts
- `flushToMemory` call in `distillSession`: passes `sessionId` (already in scope as function parameter)

### reflect.ts
- `target.addMemories` call in `flushReflectionToMemory`: passes `"reflection"` as synthetic session identifier

### hooks.test.ts + reflect.test.ts
- All `flushToMemory` test calls updated to include `"test-session"` as sessionId
- `toHaveBeenCalledWith` assertions updated to include sessionId
- `reflect.test.ts` updated to expect `"reflection"` as third argument

## Decisions Made

1. `sessionId` is required (no default) on `flushToMemory` — empty string is falsy in Python (`not ""` is `True`), which would trigger the 400 from `/add_batch` enforcement
2. `"reflection"` is the synthetic session identifier for reflection path — satisfies the non-empty string truthiness check; `source` field already disambiguates the path type if needed for filtering
