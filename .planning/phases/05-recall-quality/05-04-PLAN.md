**routes.py — domain relevance scoring:**

Add a `_domain_relevance_score(memory_text: str, query_context: str, min_factor: float = 0.6) -> float` function:
- Computes token-level Jaccard similarity between the memory text and the full query context
- The query string in /search already includes "Thread context: ..." from the `threadSummary` field — this provides the domain signal
- Tokenize both strings (lowercase, split on whitespace)
- Remove common stop words: {"the", "a", "an", "is", "in", "of", "to", "and", "or", "for", "it", "was", "be", "has", "had", "not", "are", "but", "at", "on", "with", "this", "that", "from", "by"}
- Compute: `overlap = len(query_tokens & memory_tokens) / len(query_tokens)` if query_tokens else 1.0
- Return: `max(min_factor, min(1.0, 0.6 + relevance * 0.4))` — maps [0,1] relevance into [min_factor, 1.0] score multiplier
- min_factor=0.6 means the worst penalty is 40% reduction — cross-domain results are penalized but NEVER excluded (per user decision: soft boundaries)

Add a `_apply_domain_reranking(results: list[dict[str, Any]], query: str) -> list[dict[str, Any]]` function:
- For each result in results:
  - Get memory text from `result.get("memory", "")`
  - Compute `factor = _domain_relevance_score(memory_text, query)`
  - Apply: `result["score"] *= factor` (in-place, same pattern as _apply_confidence_weight)
- Return results

Wire `_apply_domain_reranking` into the /search endpoint's response processing pipeline:
- Call AFTER noise filtering (from Plan 02) and confidence weighting
- Call BEFORE final sort and limit
- Only apply when the query contains context (check for "Thread context:" or similar marker — if query is bare, skip re-ranking since there's no domain signal)

**IMPORTANT:** The `_apply_domain_reranking` modifies scores in-place. Per Pitfall 6 from research: be aware that multiple score adjustments compound. The `minScore` filter should be applied to the ORIGINAL Qdrant score, not the adjusted score. Check whether the current code applies minScore before or after adjustments — if after, move it before (or preserve original score in a separate field for comparison).

**Test updates:**

- Test that min_factor prevents scores from dropping below 60% of original
- Test that empty query context (no Thread context) skips re-ranking (factor = 1.0)
- Test that cross-domain results are present in output (not removed)
- Test stop word removal works (common words don't inflate relevance)
