  <files>

  <action>
    Create `graph_extraction.py` — the core SimpleKGPipeline integration:

    1. **LLM adapter for OAuth compatibility:**
       The existing `llm_backend.py` has `OAuthAnthropicLLM` for Mem0. neo4j-graphrag's `AnthropicLLM` requires `api_key`. Create a `GraphRAGLLMAdapter` that:
       - For API-key backend: instantiate `neo4j_graphrag.llm.AnthropicLLM` with the API key directly
       - For OAuth backend: instantiate `neo4j_graphrag.llm.AnthropicLLM` with a placeholder API key, then replace its internal `client` with an `anthropic.Anthropic(auth_token=token)` instance (same monkey-patching pattern used in `OAuthAnthropicLLM.create()`)
       - For Ollama/none backend: return None (no graph extraction available)
       Add a `create_graphrag_llm(backend: dict)` function in graph_extraction.py that returns the appropriate LLM or None.

    2. **Schema construction:**
       Import `CONTROLLED_VOCAB` from `vocab.py` (updated in Plan 01 — no RELATES_TO). Build the schema dict:
       ```python
       {
           "node_types": ["Person", "Organization", "Place", "Concept", "Technology", "Vehicle", "Entity"],
           "relationship_types": sorted(CONTROLLED_VOCAB),
           "additional_node_types": True,
           "additional_relationship_types": False,  # Hard enforcement
       }
       ```

    3. **Pipeline initialization:**
       Create `init_pipeline(backend: dict)` that:
       - Calls `create_graphrag_llm(backend)` for the LLM
       - Creates `SimpleKGPipeline(llm=llm, driver=neo4j_driver(), schema=schema, on_error="IGNORE", perform_entity_resolution=True)`
       - Returns the pipeline instance, or None if no LLM available
       - Cache the pipeline instance at module level (reinitialize on OAuth token refresh)

    4. **Extraction function:**
       Create `async def extract_graph(text: str, backend: dict | None = None) -> dict`:
       - Get or initialize the cached pipeline
       - If pipeline is None (no LLM, no Neo4j): return `{"ok": False, "reason": "no_pipeline"}`
       - Call `await pipeline.run_async(text=text)`
       - Return `{"ok": True}` on success
       - Catch exceptions, log warning, return `{"ok": False, "reason": str(e)}`
       - This is fire-and-forget from routes — failures don't block the memory write

    5. **Batch extraction:**
       Create `async def extract_graph_batch(texts: list[str], backend: dict | None = None) -> dict`:
       - Join texts with newlines into a single string (SimpleKGPipeline's text splitter handles longer texts per research)
       - Call extract_graph with the combined text
       - This is used by /add_batch to avoid N individual LLM calls

    Create tests/test_graph_extraction.py:
    - test_schema_includes_vocab: verify schema["relationship_types"] == sorted(CONTROLLED_VOCAB) and "additional_relationship_types" is False
    - test_schema_excludes_relates_to: verify "RELATES_TO" not in schema["relationship_types"]
    - test_create_graphrag_llm_apikey: mock backend with api_key, verify AnthropicLLM created
    - test_create_graphrag_llm_oauth: mock backend with oauth, verify client patching
    - test_create_graphrag_llm_none_for_tier3: mock tier 3 backend, verify returns None
    - test_extract_graph_returns_ok: mock pipeline.run_async, verify extract_graph returns {"ok": True}
    - test_extract_graph_handles_failure: mock pipeline.run_async to raise, verify returns {"ok": False}
    - test_init_pipeline_returns_none_without_neo4j: mock neo4j_available() as False, verify returns None

    Note: Tests should mock neo4j-graphrag imports and neo4j connectivity — no live services required.
