  </files>
  <action>
    Create `backfill_relates_to.py` as a standalone migration script (run with `python backfill_relates_to.py`):

    **Script structure:**
    1. **Before stats:** Query and print current RELATES_TO count and rate:
       ```cypher
       MATCH ()-[r]->() WITH count(r) AS total
       MATCH ()-[r2:RELATES_TO]->() RETURN count(r2) AS relates_to, total, toFloat(count(r2)) / total AS rate
       ```

    2. **Fetch all RELATES_TO edges:**
       ```cypher
       MATCH (a)-[r:RELATES_TO]->(b) RETURN id(r) AS rid, a.name AS source, b.name AS target, properties(r) AS props
       ```

    3. **Checkpointing:** Write processed edge IDs to `backfill_state.json` in the sidecar directory. On restart, skip already-processed IDs. This handles crashes/interrupts.

    4. **Batch processing (50 edges per batch):**
       For each batch of RELATES_TO edges:
       - Build a prompt for the LLM: "Given source entity '{source}' and target entity '{target}', classify the relationship type. Choose from: {CONTROLLED_VOCAB list}. If no type fits with confidence, respond with 'DELETE'."
       - Use the Anthropic LLM (same backend detection as graph_extraction.py — detect_backend() from llm_backend.py)
       - For API-key: use anthropic SDK directly
       - For OAuth: use the token from llm_backend
       - Parse LLM response: either a vocab type or DELETE
       - If vocab type: CREATE new typed edge with same properties, DELETE old RELATES_TO edge
       - If DELETE or unparseable: DELETE the RELATES_TO edge (per locked decision — no quarantine)
       - Log each reclassification: `{source} -> {target}: RELATES_TO -> {new_type}` or `{source} -> {target}: DELETED`

    5. **After stats:** Print final RELATES_TO count and rate using same Cypher query.

    6. **CLI flags:**
       - `--dry-run`: Print what would be done without making changes (default mode)
       - `--execute`: Actually perform the reclassification
       - `--batch-size N`: Override batch size (default 50)
       - `--resume`: Resume from checkpoint file (default: auto-resume if checkpoint exists)

    7. **Rate limiting:** Add 0.5s delay between LLM calls to avoid rate limiting. Use asyncio for LLM calls.

    Create tests/test_backfill.py:
    - test_dry_run_does_not_modify: Mock Neo4j session, verify no CREATE/DELETE when dry_run=True
    - test_reclassification_creates_typed_edge: Mock LLM response with "KNOWS", verify CREATE query includes KNOWS
    - test_unclassifiable_edge_deleted: Mock LLM response with "DELETE", verify DELETE query runs
    - test_checkpoint_saves_progress: Mock processing, verify checkpoint file written with processed IDs
    - test_checkpoint_resumes: Create a checkpoint file, verify already-processed IDs are skipped
    - test_batch_sizing: Mock 120 edges, verify 3 batches of 50+50+20 processed

    Note: Tests mock Neo4j driver and LLM calls — no live services required.
  </action>
