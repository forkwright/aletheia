    Redesign vocab.py completely:

    1. **External vocab loading:** Add `load_vocab()` function that reads from `~/.aletheia/graph_vocab.json`. Expected JSON structure:
       ```json
       {"version": 1, "relationship_types": [...], "fallback_type": null, "normalization_log": true}
       ```
       If file not found or parse error, fall back to hardcoded defaults.

    2. **Remove RELATES_TO from everywhere:**
       - Remove RELATES_TO from CONTROLLED_VOCAB set
       - Remove "relates_to" entries from TYPE_MAP
       - Remove the `return "RELATES_TO"` final fallback in normalize_type()
       - Keep KEYWORD_MAP entries that map TO other types (those are fine)

    3. **Update normalize_type() behavior:**
       - If exact match in CONTROLLED_VOCAB → return it
       - If lowercase match in TYPE_MAP → return mapped type
       - If normalized (underscores/hyphens) match in TYPE_MAP → return mapped type
       - If keyword partial match in KEYWORD_MAP → return matched type
       - If no match found → return None (not RELATES_TO). The caller decides what to do.
       Change the return type from `str` to `str | None`.

    4. **Update TYPE_MAP:** Change entries that currently map to RELATES_TO:
       - `"is": "RELATES_TO"` → remove this entry (too vague to map meaningfully)
       - `"is_a": "RELATES_TO"` → remove this entry
       - `"relates_to": "RELATES_TO"` → remove this entry

    5. **Module-level CONTROLLED_VOCAB:** Set `CONTROLLED_VOCAB = load_vocab()` at module level (returns frozenset). This is the same pattern as current code but using the new loader.

    6. **Update GRAPH_EXTRACTION_PROMPT in config.py:** Remove RELATES_TO from the prompt's relationship type list. Remove the line "Use RELATES_TO as fallback when no specific type fits." The prompt should list only the controlled vocabulary types and instruct the LLM to select the most specific applicable type, skipping the relationship if none fits.

    - test_normalize_unknown_returns_none: "FOOBAR_NONSENSE" → None

    - test_load_vocab_fallback_on_missing_file: with no file present, load_vocab returns hardcoded defaults
    - test_load_vocab_fallback_on_corrupt_file: with invalid JSON, load_vocab returns hardcoded defaults
