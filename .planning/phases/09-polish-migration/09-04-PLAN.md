Read ToolStatusLine.svelte, ThinkingStatusLine.svelte, ToolPanel.svelte, and ThinkingPanel.svelte before writing anything. Match their CSS conventions, Svelte 5 syntax, and prop patterns exactly.

**PlanningStatusLine.svelte** — the pill button:
- Props: `projectId: string`, `state: string`, `activeWave: number | null`, `onclick: () => void`
- Derives status text from state:
  - `state === "executing" && activeWave !== null` → `"Wave ${activeWave + 1} running"`
  - `state === "verifying"` → `"Verifying phase"`
  - `state === "complete"` → `"Planning complete"`
  - `state === "blocked"` → `"Blocked — needs input"`
  - `state === "phase-planning"` → `"Planning phases"`
  - default → capitalize first letter of state
- Visual: matches `tool-status-line` / `thinking-status-line` pill pattern — `border-radius: var(--radius-pill)`, `display: inline-flex`, `gap: 6px`, `padding: 4px 10px`
- Active states (executing, verifying, phase-planning, researching, roadmap, requirements, questioning): show `Spinner` component from `../shared/Spinner.svelte` and left border `border-left: 3px solid var(--status-active)` (same as ThinkingStatusLine active state)
- Terminal states (complete, blocked, abandoned): no spinner, different border color — use `var(--status-complete)` for complete, `var(--status-error)` for blocked/abandoned
- Button has `onclick` prop that opens the panel

**PlanningPanel.svelte** — the right-pane panel:
- Props: `projectId: string`, `onClose: () => void`
- Same outer structure as ThinkingPanel.svelte: `width: 380px`, slide-in animation, `border-left: 1px solid var(--border)`, close button in header
- Header: "Planning Execution" title + close button
- Polling via `$effect`:
  ```typescript
  $effect(() => {
    if (!projectId) return;
    fetchSnapshot(); // immediate call
    const iv = setInterval(fetchSnapshot, 2500);
    return () => clearInterval(iv); // REQUIRED cleanup
  });
  ```
- `fetchSnapshot()` calls `GET /api/planning/projects/${projectId}/execution` and sets local `snapshot` state
- Body: renders `snapshot.plans[]` — for each plan: name, status badge (color-coded: pending=gray, running=blue, done=green, failed=red, skipped=gray, zombie=orange)
- Show `snapshot.state` as subtitle below header
- Handle loading state (snapshot is null before first fetch): show a spinner
- Handle error state (fetch fails): show "Unable to load status" message — do not crash

Follow pitfall 5 from RESEARCH.md: `$effect` with setInterval MUST return `() => clearInterval(iv)` or the interval leaks on unmount.

API response shape (ExecutionSnapshot):
```typescript
interface ExecutionSnapshot {
  projectId: string;
  state: string;
  activeWave: number | null;
  plans: PlanEntry[];
  activePlanIds: string[];
  startedAt: string | null;
  completedAt: string | null;
}
interface PlanEntry {
  phaseId: string;
  name: string;
  status: string; // pending|running|done|failed|skipped|zombie
  waveNumber: number | null;
  startedAt: string | null;
  completedAt: string | null;
  error: string | null;
}
```
