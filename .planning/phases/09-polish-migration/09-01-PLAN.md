**## Problem**
Why existing session-scoped tools (plan_create, plan_propose) fall short: they lose state across sessions, cannot orchestrate multi-agent work, and have no persistence. What Dianoia enables: persistent multi-phase planning with wave-based parallel execution, goal-backward verification, and human-in-loop checkpoints — all state surviving arbitrary session boundaries.

**## Design**

### Principles
- Persistence over re-derivation: all planning state in SQLite, survives restart
- State machine as the single source of truth: all state changes go through DianoiaOrchestrator
- Constructor injection: orchestrators accept db and dispatchTool, no global state
- Async-safe: each phase runs in wave-parallel subagent sessions; the planning FSM coordinates, not executes
