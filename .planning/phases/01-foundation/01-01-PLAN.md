**Methods to implement** (all multi-step mutations use `db.transaction()`):

*Projects:*
- `createProject(opts: { nousId, sessionId, goal, config })`: computes contextHash via SHA-256 of `${goal}|${nousId}|${createdAt}` truncated to 16 hex chars. Wraps INSERT in `db.transaction()`. Returns `PlanningProject`.
- `getProject(id)`: returns `PlanningProject | undefined`.
- `getProjectOrThrow(id)`: returns `PlanningProject`, throws `PlanningError` with code `PLANNING_PROJECT_NOT_FOUND` if missing.
- `listProjects(nousId?: string)`: returns `PlanningProject[]`, optionally filtered by `nous_id`.
- `updateProjectState(id, state)`: UPDATE `state` and `updated_at`. Throws `PLANNING_PROJECT_NOT_FOUND` if missing. Uses transaction.
- `updateProjectConfig(id, config)`: UPDATE `config` (JSON-serialized) and `updated_at`. Throws `PLANNING_PROJECT_NOT_FOUND` if missing. Uses transaction.
- `deleteProject(id)`: DELETE with CASCADE. Returns `void`.

*Phases:*
- `createPhase(opts: { projectId, name, goal, requirements, successCriteria, phaseOrder })`: INSERT. Wraps in transaction. Returns `PlanningPhase`.
- `getPhase(id)`: returns `PlanningPhase | undefined`.
- `getPhaseOrThrow(id)`: throws `PLANNING_PHASE_NOT_FOUND`.
- `listPhases(projectId)`: returns `PlanningPhase[]` ordered by `phase_order`.
- `updatePhaseStatus(id, status)`: UPDATE `status` + `updated_at`. Transaction.
- `updatePhasePlan(id, plan)`: UPDATE `plan` (JSON-serialized) + `updated_at`. Transaction.

*Requirements:*
- `createRequirement(opts: { projectId, phaseId, reqId, description, category, tier })`: INSERT. Transaction. Returns `PlanningRequirement`.
- `listRequirements(projectId)`: returns `PlanningRequirement[]`.

*Checkpoints:*
- `createCheckpoint(opts: { projectId, type, question, context })`: INSERT (no decision yet). Returns `PlanningCheckpoint`.
- `resolveCheckpoint(id, decision)`: UPDATE `decision`. Transaction. No `updatedAt` on checkpoints table.
- `listCheckpoints(projectId)`: returns `PlanningCheckpoint[]`.

*Research:*
- `createResearch(opts: { projectId, phase, dimension, content })`: INSERT. Returns `PlanningResearch`.
- `listResearch(projectId)`: returns `PlanningResearch[]`.
