**1. Create dianoia/orchestrator.ts**

New file. Logger name: `"dianoia:orchestrator"`. Import order: node modules → external (better-sqlite3 type) → internal (createLogger, eventBus, PlanningStore, transition, PlanningConfigSchema).

```typescript
// DianoiaOrchestrator — single state driver for all planning entry points
import { createLogger } from "../koina/logger.js";
import { eventBus } from "../koina/event-bus.js";
import { PlanningStore } from "./store.js";
import { transition } from "./machine.js";
import type Database from "better-sqlite3";
import type { PlanningConfigSchema } from "../taxis/schema.js";
import type { PlanningProject } from "./types.js";

const log = createLogger("dianoia:orchestrator");

const ACTIVE_STATES = new Set([
  "idle", "questioning", "researching",
  "requirements", "roadmap", "phase-planning",
  "executing", "verifying", "blocked",
]);

export class DianoiaOrchestrator {
  private store: PlanningStore;

  constructor(db: Database.Database, private defaultConfig: PlanningConfigSchema) {
    this.store = new PlanningStore(db);
  }
```

**handle(nousId, sessionId): Promise<string>** — the main entry point called by /plan command and CLI.

Behavior (per locked decisions in CONTEXT.md):
1. Find active project: `listProjects(nousId).find(p => ACTIVE_STATES.has(p.state))`
2. If active project found AND its `config` has `pendingConfirmation: true` — this is the resume-confirmation answer turn; call `confirmResume(project, nousId, sessionId, …)` (handled separately below; for `handle()` we just return early noting it's not the right path — see note)
3. If active project found (no pending confirmation): update config to set `pendingConfirmation: true` via `store.updateProjectConfig(...)`, return confirmation question: `Still working on "${project.goal || 'your project'}"? (yes to resume, no to start fresh)`
4. If no active project: create new project via `store.createProject({ nousId, sessionId, goal: "", config: this.defaultConfig })`, transition idle → questioning via `store.updateProjectState(id, transition("idle", "START_QUESTIONING"))`, emit `planning:project-created`, return: `"Starting a Dianoia planning project. First: what are you building?"`

Note: `handle()` is called by the /plan command. The resume yes/no response arrives as a regular chat turn — the `turn:before` handler (Plan 02-03) detects the pending confirmation flag and routes to `confirmResume()` / `startFresh()`.

**confirmResume(projectId, nousId, sessionId, answer: string): Promise<string>**

- Clear `pendingConfirmation` flag from project config via `store.updateProjectConfig(projectId, { ...project.config, pendingConfirmation: false })`
- If `answer.toLowerCase().includes("yes")` or `answer.toLowerCase().includes("y")`:
  - emit `planning:project-resumed` with `{ projectId, nousId, sessionId }`
  - log.info resume
  - return `"Resuming your planning project. You're in the ${project.state} phase."`
- If no (any other response):
  - call `this.abandon(projectId)`
  - create new project (same as fresh start path in `handle()`)
  - return the first question string

**abandon(projectId: string): Promise<void>**

- `store.getProjectOrThrow(projectId)` — throws PlanningError if not found
- `store.updateProjectState(projectId, "abandoned")` — direct write, no FSM transition needed (abandoned is a valid terminal, but only ABANDON event reaches it from any state; use `transition(project.state, "ABANDON")`)
- log.info

**getActiveProject(nousId: string): PlanningProject | undefined**

- `listProjects(nousId).find(p => ACTIVE_STATES.has(p.state))`

**hasPendingConfirmation(project: PlanningProject): boolean**

- `return (project.config as Record<string, unknown>)["pendingConfirmation"] === true`
