  <behavior>
**Function signature:** `export function detectPlanningIntent(text: string): boolean`

**TRUE positives (should return true):**
- "I want to build a new project management SaaS" — "build"+"app/system/service" pattern
- "Help me plan the architecture for a distributed caching system" — "help me plan"+"architecture"
- "I need to design a microservices platform from scratch" — "design"+"system/platform"
- "Let's create a requirements doc for my new mobile app" — "create"+"requirements"
- "I'm starting a new project — can you help me map out the phases?" — "new project"+"phases"
- "We need to build out a roadmap for the analytics pipeline" — "roadmap"+"pipeline"
- "What are the requirements for building a real-time notification system?" — "requirements"+"building"

**FALSE positives (should return false):**
- "Can you fix this bug in my auth middleware?" — debugging, no planning signal
- "How do I center a div in CSS?" — simple question
- "Read this file and summarize it" — file operation
- "Run the tests and show me the output" — task execution
- "What is the difference between REST and GraphQL?" — conceptual question
- "Update the README" — simple edit task
- "Can you explain how this function works?" — code explanation
- "I'm building momentum on this task" — "building" in non-project context
- "Design a button that looks like this" — UI task, not architecture
- "Create a new file called config.ts" — file creation, not project planning

**Detection logic (Claude's discretion per CONTEXT.md):**

Detect UNAMBIGUOUS planning intent. Two or more planning signals must co-occur:
1. Action verb: plan, design, architect, build, create, develop, implement, start, launch
2. Subject noun: project, system, app, application, service, platform, product, tool, pipeline
3. Complexity indicator: roadmap, requirements, phases, architecture, milestones, scope, design doc

OR explicit command-style phrases (single signal sufficient):
- "help me plan", "start a project", "new project", "planning project"
- "/plan" appearing anywhere in text

Do NOT match on single generic verbs ("build", "create") without a project-scale noun. "Build a function" is not planning intent. "Build a platform" is.

**Why this approach:** The research identifies sensitivity as "detect unambiguous planning intent (new project, complex multi-phase work), NOT simple tasks." Requiring 2+ signals reduces false positives on everyday messages.
