  <action>
    Create `infrastructure/memory/scripts/purge-qdrant-orphans.py` — a standalone one-time cleanup script.

    **Behavior:**
    1. Connect to Qdrant (default: localhost:6333, configurable via `--host` and `--port` args)
    2. Accept `--collection` arg (default: the collection name used in the sidecar — check `routes.py` or `app.py` for the actual collection name)
    3. Scroll through all points in the collection (batch size 200, `with_payload=True, with_vectors=False`)
    4. For each point, check payload for required fields: `session_id`, `source`, `agent_id`
    5. If any required field is missing or empty/None, mark as orphan
    6. During scroll, track and print a distribution of `source` values seen across orphans (for diagnostic insight per Research open question #2)
    7. **Default mode (dry-run):** Print orphan count, source value distribution, and sample of 5 orphan point IDs. Do NOT delete.
    8. **With `--execute` flag:** Delete all identified orphans via `client.delete()`. Log orphan IDs to a file `purged-orphans-{timestamp}.log` in the same directory before deletion.
    9. Print summary: total scanned, orphans found, orphans deleted (if --execute)

    **CLI interface:**
    ```
    python purge-qdrant-orphans.py                          # dry-run
    python purge-qdrant-orphans.py --execute                # actually delete
    python purge-qdrant-orphans.py --host 192.168.0.29 --port 6333 --execute
    ```

    Use `argparse` for CLI args. Use `qdrant_client` Python package (already installed in sidecar environment).

    Add a shebang line `#!/usr/bin/env python3` and a one-line module comment.
  </action>
