    1. Run `npx oxlint src/mneme/` to identify linter-flagged unused code. Also run `npx tsc --noEmit` and look for unused variable/import warnings (noUnusedLocals and noUnusedParameters are enabled per tsconfig).

    2. Read every file in `infrastructure/runtime/src/mneme/` and identify:
       - **Unused imports:** Imports not referenced anywhere in the file
       - **Unused exports:** Functions/types exported from mneme modules but not imported by any other module (use `grep -r "from.*mneme" src/` to find all consumers)
       - **Unreachable branches:** Switch cases that can never match, if-branches guarded by always-true/always-false conditions, catch blocks for exceptions that cannot be thrown
       - **Bypassed code paths:** Functions that exist but are never called from any code path (dead methods on SessionStore, etc.)
       - **Stale migration code:** Check if any migration entries reference tables or columns that were later dropped — these migrations still need to exist (idempotent) but any runtime code referencing dropped columns is dead

    3. For each identified dead code item:
       - If it's clearly dead (unused import, unreachable branch): Remove it
       - If it's a commented block explaining an architectural decision (e.g., "// We don't use X because Y"): Preserve it per user decision
       - If uncertain whether something is dead: Check all importers and callers via grep before removing

    4. After all removals, verify:
       - `npx tsc --noEmit` still passes
       - `npx vitest run src/mneme/` still passes
       - `npx oxlint src/mneme/` is clean (no new warnings)

    Do NOT remove test files or test utilities — only production code in mneme/.
