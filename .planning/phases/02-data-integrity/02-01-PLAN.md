<objective>
Crash-safe distillation locking and atomic transaction rollback

Purpose: Replace the in-memory `activeDistillations` Set (which evaporates on process crash, leaving sessions permanently locked) with a SQLite-backed lock table. Bundle the four separate distillation mutation calls into a single atomic transaction so mid-distillation crashes leave no partial state.

Output: SQLite lock table with startup stale-lock scan, atomic `runDistillationMutations()` method with single-retry, tests for both behaviors.
</objective>

       - `activeDistillations.has(sessionId)` → `!store.acquireDistillationLock(sessionId, nousId)` (acquireDistillationLock returns false if already locked)

    3. Replace the sequence of individual store mutation calls (appendMessage, markMessagesDistilled, recordDistillation, updateLastDistilledAt, recordDistillationLog) with a single call to `store.runDistillationMutations(opts)`.     4. Add single-retry logic around the `runDistillationMutations` call per user decision:
       ```typescript
       try {
         store.runDistillationMutations(opts);
       } catch (firstErr) {
         log.warn("Distillation mutations failed (attempt 1)", { sessionId, error: firstErr });
         try {
           store.runDistillationMutations(opts);
         } catch (secondErr) {
           log.error("Distillation mutations failed after retry", { sessionId, error: secondErr });
           // Do not rethrow — next scheduled distillation handles it

       }
       ```
