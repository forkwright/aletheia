# Website Information Architecture Mapping
Systematically extract comprehensive information about a website's structure, content, and offerings by fetching key pages.

## When to Use
When you need to understand a website's complete organization, products, values, and content structure. Useful for business research, competitive analysis, or gathering detailed information about an organization's offerings and philosophy.

## Steps
1. Fetch the homepage to understand the site's core identity and navigation structure
2. Fetch main section pages (journal, philosophy, materials, dyes) to understand content pillars
3. Fetch product/offerings index pages to identify what's available
4. Fetch individual product pages to gather detailed information (pricing, descriptions, specifications)
5. Fetch supporting pages (lexicon, materials sourcing) to understand deeper context and values

## Tools Used
- web_fetch: Used to retrieve and parse HTML content from each URL, capturing both structural information and textual content to build a complete picture of the website
