# Repository Status Assessment and Draft Document Discovery

Identify open issues, pull requests, and locate draft or in-progress specifications in a project repository.

## When to Use
When you need to audit a project's current state, find pending work items, or locate documents with specific status markers (Draft, Approved, In Progress, etc.) for review or completion.

## Steps
1. Navigate to the project repository directory
2. Query for open issues using version control system commands
3. Query for open pull requests using version control system commands
4. Search for files containing specific status markers in a designated documentation directory
5. Extract and display the header/title information from matching files to understand their scope

## Tools Used
- exec: Execute shell commands to navigate directories, run git/GitHub CLI commands, and search files with grep
